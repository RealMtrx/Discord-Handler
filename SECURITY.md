# Security Policy

## Supported Versions

All implementations in the `main` branch receive security updates. Older releases may receive patches on a best-effort basis.

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** open a public issue. Instead, send a direct message to the repository owner or email `mtrxturki@gmail.com`.

You should receive a response within 48 hours. If you do not, please follow up.

## What to Include

- A clear description of the vulnerability
- Steps to reproduce it
- Potential impact
- Suggested fix (if applicable)

## Disclosure Policy

We follow a 90-day disclosure window. After a fix is deployed, we will publicly acknowledge the researcher.

## Scope

This policy covers all code within the Discord Handler organization repositories. Third-party library vulnerabilities should be reported to the respective maintainers.
